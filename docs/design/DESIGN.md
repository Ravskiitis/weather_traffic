## Implementation guidance for astro-frontend

- Reimplement the prototype idiomatically in Astro + Tailwind. Do not pixel-clone raw HTML.
- Extract design tokens (colors, font sizes, spacing, radii, shadows) into frontend/tailwind.config.mjs.
- Match visual hierarchy: KPI tiles -> AI analysis hero card -> weather + map row -> correlation chart -> incidents table.
- Maintain bilingual readiness from day one: every user-facing string lives in i18n files.

## Visual decisions

- **Style:** Clean B2B SaaS aesthetic. Light theme. Generous whitespace.
- **Accent color:** warm orange — used for primary actions, KPI highlights, and AI agent branding.
- **Secondary accent:** deep slate blue — for headers and important text.
- **Typography:** Inter or Geist (sans-serif), clear hierarchy.
- **Cards:** subtle shadows, 8-12px border radius.
- **Icons:** Lucide-style line icons, consistent stroke width.
